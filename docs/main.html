 12#The use of multiple smaller classes (ex. Player & Npc) that
 13#work with a bigger class (Game) is an example of the use
 14#of the ADVANCED DESIGN PATTERN known as FACADE. It helps
 15#in easier code readibility and organization.
 16
 17#The code uses the COMMON DESIGN PATTERN known as SINGLETON.
 18#It is implemented using a classic singleton, wherein an
 19#instance of a class is only created once (in this case, it
 20#is the Game class). Throughout the lifetime of the program,
 21#only one Game instance is needed, thus making the use of
 22#the singleton pattern useful.
 23
 24# The use of the COMMON DESIGN PATTERN known as OBSERVER
 25# allows one object to choose to be updated based on changes
 26# to another, without constantly checking for differences.
 27# In this implementation, it allows the inventory to be
 28# updated when a plant is harvested, without constantly
 29# checking via "if" statements.
